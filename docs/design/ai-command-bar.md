# AI 命令框（闪记 AI）设计规格

## 概述

闪记 AI 是一个统一的自然语言输入框，用户可以通过它在笔记中进行**增、删、改、查**全部操作。两个入口共享同一套交互模型：

| 入口 | 默认模式 | AI 触发方式 |
|------|---------|-----------|
| 主窗口命令框 | 本地关键词过滤 | `/` 前缀显式触发 AI |
| Alt+Space 快捷窗口 | AI 意图识别 | 默认走 AI，也支持 `/` 命令 |

---

## 命令体系

### `/` 子命令

输入 `/` 弹出下拉菜单，列出可用命令。用户可用键盘上下选择 + Enter 确认，或直接用鼠标点击。

| 完整命令 | 别名 | 功能 |
|---------|------|------|
| `/search` | `/s` | AI 语义搜索 |
| `/add` | `/a` | AI 解析并创建笔记 |
| `/delete` | `/d` | AI 查找并删除笔记 |
| `/edit` | `/e` | AI 查找并编辑笔记 |

### 选择流程

```
输入 / → 弹出下拉 (4项)
        第一项默认高亮
        ↑↓ 移动  Enter 选择  Esc 关闭

输入 /s → 下拉过滤到 /search
          Enter 选中 → 切换到搜索模式

输入 /a → 下拉过滤到 /add
          Enter 选中 → 切换到创建模式
```

## 交互模型

### 主窗口 — 正常模式（无 `/` 前缀）

```
用户输入关键词
  │
  ├── 实时本地过滤卡片（标题/内容/标签）
  │
  ├── @ 触发分类筛选
  │
  └── 提示: "关键词过滤 · @分类筛选 · / 切换AI模式"
```

**行为：**
- 纯本地匹配，不走 AI
- 不创建、不修改、不删除
- Enter = 触发 AI 语义搜索（输入含空格时）
- `@category` 筛选卡片

### 主窗口 — AI 模式（`/` 前缀）

```
用户输入 /<命令> <内容>
  │
  ├── 有显式子命令 (/search, /s, /add 等)
  │     └── 直接执行对应操作，内容送 AI
  │
  └── 无显式子命令 (/ 自然语言)
        └── AI 先识别意图 → 展示意图标签 → 用户确认 → 执行
```

**示例：**

```
/s docker compose down
  → AI 语义搜索，返回匹配卡片

/a sk-xxx 这是我的 deepseek key
  → AI 解析内容 → 标题/类型/标签 → 创建笔记

/d 昨天那条 docker 命令
  → AI 定位笔记 → 展示匹配项 → 用户确认 → 删除

/e openai key 改成测试环境
  → AI 定位笔记 → 展示编辑预览 → 用户确认 → 修改

/ 帮我找 docker 相关命令
  → AI 识别意图 → 展示 "🔍 搜索" → 用户确认 → 执行
```

**无显式子命令时，AI 识别意图后展示确认：**

```
┌───────────────────────────────────────────┐
│ ⚡ / 帮我删掉 docker 那条命令              │
│                                           │
│ AI 识别意图: 🗑 删除                       │
│ 匹配: >_ Docker 清理所有悬空镜像           │
│                                           │
│ [Enter 确认] [Esc 取消]                   │
└───────────────────────────────────────────┘
```

### Alt+Space — 默认 AI 模式

```
所有输入默认走 AI 识别意图
  │
  ├── 普通输入 → AI 判断是搜索还是创建
  │     "docker compose"  → 搜索（有匹配）或 创建（无匹配）
  │     "sk-xxx my key"   → 创建（识别为 API Key）
  │
  ├── / 命令 → 同主窗口 AI 模式，显式控制
  │
  └── 结果展示为紧凑列表（3-5条）
        Enter 选中项 → 执行主操作（复制/打开）
```

**与主窗口的区别：**
- 默认全部走 AI（不需要 `/` 前缀）
- 结果列表紧凑，不展示卡片墙
- 创建时静默保存，不弹窗
- `/` 命令同样可用（显式控制更高效）

---

## 四个操作总结

| 操作 | 主窗口正常模式 | 主窗口 / AI 模式 | Alt+Space | 卡片上 |
|------|-------------|-----------------|-----------|-------|
| **查** | 关键词本地过滤 | `/s` 语义搜索 | 输入即搜 | — |
| **增** | — | `/a` 创建 | 输入即创建 | — |
| **删** | — | `/d` + 确认 | — | 删除按钮 |
| **改** | — | `/e` + 确认 | — | 编辑按钮 |

> 卡片上的编辑/删除按钮作为保底，不需要命令框也能直接操作。

## 下拉菜单交互规范

`/` 命令下拉与 `@` 分类下拉共用同一套交互模式：

- 触发时第一项默认高亮
- `↑` `↓` 移动高亮
- `Enter` 确认选择
- `Esc` 关闭下拉
- 继续输入过滤匹配项
- 高亮项超出可视区时自动滚动跟随
- 鼠标 hover 同步高亮，click 确认选择
