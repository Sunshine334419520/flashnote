## 三、架构问题汇总

### IPC 通道：27 个定义 vs 20 个实际使用

```
已定义未使用：
  - search:query（renderer 用 notes.list 代替）
  - event:ai-classification-complete（从未广播）
  - event:settings-changed（从未广播）
  - ai:parse（preload 暴露但 renderer 未调用）
  - WINDOW_SHOW_MAIN（错误委托给 showSettingsWindow）

使用中但 MainView 未订阅：
  - event:note-deleted
```

### 组件：过渡期产物

当前 CardWall 为主界面的设计是一次架构转向——从三栏布局（AppShell → Sidebar + NoteListPanel + NoteDetailPanel）转向卡片墙。但旧组件未清理，导致 6 个组件成为死代码。

### 数据流断裂

- `QuickCapture` 绕过 Zustand store 直接调用 IPC
- `NoteDetailPanel` 是唯一调用 `updateNote`/`deleteNote` 的组件，但它未被渲染
- 卡片点击设置了 `selectedNoteId` 但无后续 UI 响应
